line-height继承问题

Posted on 2018-08-21 | In css

最近在开发中遇到了一个line-height的值影响当前页面布局的问题,具体的如下图所示：

在没有给子元素设置任何的属性的时候，子元素会和font-size,color这些属性继承父元素设置的line-height属性，会根据line-height的值自动撑开元素

给父元素设置默认值会，子元素就会回归到正常的状态，但是在有些情况下父元素是需要设置line-height属
性的，子元素如果不想继承父元素的line-height可以自行设置line-height的属性值

但是如果将line-height设置为0就会出现容器内的内联元素会在父元素的上方溢出,如下图所示：

如果是多个内联元素就会出现重叠的现象，如下图所示：

当前容器的宽度不足以放下所有内容时，里面的内容就会折行。当父元素的line-height设置为0时内联元素>的的line-height会继承父元素的，从父元素的左上角为起点，内联元素的中间点为起点开始计算。超出父元素
的部分计算为负值。为在父元素里面的部分为正值。将元素中间放在父元素的开始，上下距离相加等于父元素>的line-height值，绘制子元素的位置。当子元素为内联元素的时候，折行的部分算作一个新的内容，同上计算
line-height值，所以会出现重叠的现象

当子元素为块级元素并设置了子元素的高度

内联元素永远会基于父元素的起点开始计算line-height的值,如果不设置内联元素父元素的高度，会出现和>内联元素一样的效果，全部堆积到一块了。
