new Object 和 Object.create的区别

Posted on 2018-05-02 | In javascript

最近正在看es6的class语法，看到了一个示例用了Object.create(),就想到为什么不用new Object于是就有了看一下Object.create()和new Object的想法。

为了方便书写 Object.create() 用create代替，new Object用new代替

创建了一个名为Base的构造函数，分别用 new 和 create创建了两个实例。打印的结果出乎意。o1实例的Base类 o2是Function类

o1的__proto__指向的是Base类的prototype,Base类的__proto__指向的是Object这个基类的原型.

o2指向的是Function的prototype, Function本身就是基类.

那么这两个具体的区别是什么那？

为Base构造函数的prototype增加一个say方法，o1、o2实例分别执行这个方法。
o1.say() 执行的结果是 hello ，o2.say()抛错说 o2.say 不是一个函数

然后我分别打印了两个实例的__proto__，发现两个实例的__proto__的指向不是一样的，o1的指向是Base构造函数的原型，而o2的指向是构造函数本身。由此就可以明白了为什么o2.say()不是一个函数了，o2的prototype是Base构造函数，而Base的__proto__的指向是Function这个基类

这个是我分析后画出来的原型图，通过这个原型图可以看出，o1实例指向的是Base这个构造函数的原型，o2指向的是这个构造函数，o1继承了Baseprototype上的属性和方法，而o2直接通过__proto__查找Base的所属类的原型也就是Function,并没有继承任何方法。

为什么会出现这种情况那？

分析一下Object.create的实现 方法

从Object.create的实现方式可以看出 ，实现方式是先创建一个构造函数，然后让这个对象的原型等于传递进来的方法，然后在new一个新的实例。问什么会和 new 创建的方法不一样的，只要是因为新的实例的原型指向的是构造函数，并不是构造函数的原型

写什么技术之类的啦。写写怎么搭建hexo博客吧

  • node 必备
  • hexo 安装 npm install -g hexo-cli

第一步初始化hexo博客

这个操作是在本地搭建一个hexo博客，目前操作还不涉及到部署到github

  • 这个是初始化一个博客的命令

  • 这个是生成静态文件的命令

  • 这个是启动服务的命令

博客初始化完成

服务运行起来，在浏览器的地址栏里面输入127.0.0.1:4000，就会看到你的博客

到目前为止你已经初始化完成了一个博客，但是里面的内容还是hexo自己的内容hello world，接下来就是更换自己喜欢的主题，建立自己的第一篇博客啦。

更换主题

_config.yml 是hexo的配置文件，可以在里面更改基本的配置信息，现在主要用的是主题的更换。首先我们需要下载一个博客主题。因为next的star比较多，所以这里就拿next为例。

  • 首先在当前博客的根目录下运行next代码的克隆命令
  • 然后在根目录下的_config.yml进行主题的更改,将themes改为next

这里不是next默认的主题如果想更换next自身的主题可以去themes/next/_config.yml更换,更改注释即可

发布第一篇文章

文章的是以markdown的形式编写的。编写完成保存文件后，需要生成静态文件也就是把markdown转换成网页，生成静态文件后，网站的源码就会保存在scaffolds文件夹下
执行hexo s 访问本地4000端口就可以访问你的第一篇博客了。

将博客部署到github上

  • 创建一个新的项目

  • 根据格式创建新的项目(因为我已经创建过这个仓库了，所以会有警告，声明一下，创建的仓库必须是自己github名字，不然部署到github上不会出现样式)

  • 创建完成后点击 setting 选择master branch 点击save

  • 将你的github创建的仓库地址复制到根目录下的 _config.yml

  • 这是你需要一个插件hexo-deployer-git

  • 部署完成之后，访问github setting里面提供的page地址就可以看到你的博客了

  • 如果你有域名的话可以将域名和github关联起来

    • 首先解析你的域名

      我用的是阿里云，其它的域名解析也是一样的

    • 在source文件夹下创建CNAME文件将域名写到文件夹下

    • 运行构建


访问你的域名现在就可以看到你的博客文章啦
